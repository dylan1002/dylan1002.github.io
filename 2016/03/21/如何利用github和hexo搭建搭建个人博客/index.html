如何利用github和hexo搭建搭建个人博客

发表于 2016-03-21   |  

前记：

出于兴趣从网上找到一些通过GITHUB搭建个人博客的文章，看完之后觉得挺有意思的，所以决定自己也动手搭建一个属于自己的博客，同时也希望利用其对自己在WEB前端技术学习过程中遇到的问题和难点进行记录，希望它能伴随我的成长，我的电脑是WIN8系统，搭建过程中会有一些网上其他文章所没有提到的地方，我会详细提出。

正文：

至于为什么使用github网上有很多资料，在此我就不赘述了。hexo和jekyll都是静态博客框架，他们的区别是一个用 node-js，一个用 Ruby ,jekyll 你把原文上传 github，可以直接生成博客，也可以用在线编辑器处理，而hexo 是本地生成 html再上传。我对比一下选择了hexo，因为node比较容易安装，而且 JS 的速度比 Ruby 更快。

1.使用到的工具：

  1. Git
  2. Node.js

2.安装和使用hexo：

首先确认你已经安装好上面提到的工具。

I.安装hexo

在电脑中建立一个名字叫「Hexo」的文件夹（比如我建在了G:\Hexo），然后在此文件夹中右键打开Git Bash。执行下面的命令，即可完成 Hexo 的安装：
$ npm install -g hexo-cli

II.初始化hexo

在完成hexo的安装之后，输入以下命令对hexo进行初始化，初始化结束之后，打开「Hexo」文件夹，你会发现里面已经生成许多文件，文件目录如下（截自Hexo官网）：

III.生成文件的功能

  • _config.yml 网站的配置信息，您可以在此配置大部分的参数（特别重要）
  • themes 存放主题的文件夹

  • source 博客资源文件夹，也就是以后存博文的地方
    其他更详细的功能解释，请到官方文档查看.

    IV.更换主题。

    1 首先，到官网找到自己喜欢的主题。https://github.com/hexojs/hexo/wiki/Themes
    2 然后利用以下代码对主题进行克隆（以light为例）
    $ git clone https://github.com/hexojs/hexo-theme-light.git
    3 然后对主题进行配置，依次输入
    $ cd themes/light
    $ git pull
    然后你会发现themes目录里面会多出一个light文件，说明主题配置成功。
    4 现在可以依次输入：
    $ hexo g
    $ hexo s

在本地浏览器中输入localhost:4000 即可在本地看到你的个人博客效果 噔噔噔噔，是不是激动= =！。

3.利用GitHub使你的个人博客让所有人都能看到

  1. 首先，到官网注册一个GitHub帐号,虽然是全英文但是应该难不倒各位看官吧，不多提了。
  2. 配置和使用GitHub
    本地git项目与远程的github建立联系呢？它们之间的密匙是什么呢？那就是SSH key。
    （1）首先我们需要检查你电脑上现有的ssh key：
    $ cd ~/.ssh
    如果显示如下

    说明你的电脑是第一次使用git。
    （2）生成新的SSH Key：
    $ ssh-keygen -t rsa -C "邮件地址@youremail.com"
    这里回车就好

    然后系统会要求你输入密码，如果直接按回车那上传的时候就不需要密码。这样可能会被别人乱动你的项目
    显示以下界面说明你已经成功配置SSH key

    （3）添加 SSH Key 到 GitHub：
    首先
    win7:打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
    注意啦，win8 SSH Key存在这里C:\Users\hp-pc.ssh\id_rsa.pub
    然后登陆github系统，右上角Settings，把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了。


    （我在这里出现了较多问题，如果你也一样可以私我）
    接下来测试一下配置是否成功，输入以下代码：
    $ ssh -T git@github.com
    如果是下面的反馈：
    The authenticity of host 'github.com (207.97.227.239)' can't be established. RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48. Are you sure you want to continue connecting (yes/no)?
    输入yes、然后会看到：
    Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access.
    说明配置已经成功
    接下来设置一下你的个人信息（很重要，上传的时候GitHub 会用这些信息来做权限的处理）：
    $ git config --global user.name "dylan1002"//用户名
    $ git config --global user.email "imland@outlook.com"//填写自己的邮箱
    1. 把本地博客同步到GitHub
      接下来是最后一步咯！将你的个人博客上传到Github上去。。

      1.首先在GitHub上创建一个仓库repository，注意仓库名称必须为username.github.io，一定要是这个格式哦，username就是你注册的用户名。

      4.接下来右键之前创建的「Hexo」的文件夹，输入以下命令：
      $ npm install hexo-deployer-git --save
